# Components overview

## Таблиця компонентів

| Назва компонента | Призначення | Основні функції (3–5 булетів) | Інтерфейс / Протокол | Дані (вхід / вихід) | Технології |
|------------------|-------------|--------------------------------|----------------------|---------------------|------------|
| **frontend-web** | Презентаційний рівень. Єдиний веб-клієнт (React SPA), що відкривається в браузері на десктопі та мобільних пристроях. | • Відображення каталогу українських фільмів (списки, пошук, фільтри).<br>• Перехід на сторінку деталей фільму.<br>• Надсилання HTTP/REST-запитів до бекенду.<br>• Обробка станів завантаження/помилок.<br>• (Опційно) UI для входу/реєстрації користувача. | **HTTPS / REST (JSON)** до `backend-api`. | Вхід: дії користувача, конфігурація клієнта.<br>Вихід: HTTP-запити на `/api/...`, локальний стан додатка. | React, JavaScript/TypeScript, HTML, CSS. |
| **backend-api** | Прикладний рівень. Серверна бізнес-логіка та REST-API, єдина точка входу для клієнта. | • Обробка запитів `/api/movies`, `/api/movie/:id` тощо.<br>• Агрегація даних з TMDB API та MongoDB.<br>• Фільтрація контенту до українських фільмів.<br>• Валідація вхідних даних та формування JSON-відповідей.<br>• (Опційно) автентифікація/авторизація користувачів. | Вхід: **HTTPS / REST** від `frontend-web`.<br>Вихід: **HTTPS / REST (JSON)** до клієнта; з’єднання з MongoDB; HTTP-запити до TMDB. | Вхід: параметри запитів, дані користувачів, параметри пошуку.<br>Вихід: JSON-об’єкти (списки/деталі фільмів, статуси операцій). | Node.js, Express, HTTP-клієнт (axios/fetch). |
| **db-core** | Рівень даних. Основне сховище внутрішніх даних застосунку. | • Зберігання користувачів (якщо є логін).<br>• Зберігання «Обраного», відгуків, рейтингів.<br>• (Опційно) кеш TMDB-даних для прискорення відповідей.<br>• Підтримка індексів та базових обмежень. | З’єднання з `backend-api` через драйвер MongoDB. | Вхід: запити на читання/запис документів (insert, find, update, delete).<br>Вихід: документи колекцій (users, favorites, reviews, cached_movies тощо). | MongoDB, Mongoose (або інший ODM). |
| **tmdb-gateway** | Логічний компонент інтеграції з TMDB API як зовнішнім сервісом. | • Формування HTTP-запитів до TMDB API.<br>• Відбір українських фільмів згідно з параметрами пошуку.<br>• Мапінг JSON-відповіді TMDB на внутрішні моделі (Movie).<br>• (Опційно) кешування результатів у `db-core`. | **HTTPS / REST (JSON)** до TMDB API, виклики з `backend-api`. | Вхід: параметри пошуку/каталогу.<br>Вихід: дані TMDB, нормалізовані до внутрішніх DTO. | TMDB API, axios/fetch, зберігання API-ключа у змінних оточення. |
| **observability-ci** (майбутній) | Інфраструктурний компонент для логування, моніторингу та CI/CD. | • Логування запитів і помилок `backend-api`.<br>• Автоматичний запуск тестів і збірки при змінах у репозиторії.<br>• (Опційно) деплой бекенду/фронтенду на сервер/хмару.<br>• Базовий моніторинг доступності та часу відповіді. | Інтеграція з GitHub, середовищем виконання сервера, сервісами моніторингу. | Вхід: події з репозиторію, логи, метрики.<br>Вихід: звіти CI, алерти, оновлені версії застосунку. | GitHub Actions / GitLab CI, логер для Node.js, (опційно) Prometheus, Grafana. |
