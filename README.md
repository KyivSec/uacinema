# Components

## Overview

Система — це WebДодаток для пошуку українських фільмів з публічним інтерфейсом для користувачів та простим BackendAPI з БазоюДанихФільмів і інтеграціями з зовнішніми платформами перегляду.

---

| Назва компонента         | Призначення                                                                 | Основні функції                                                                                  | Інтерфейс / Протокол      | Дані (вхід / вихід)                                                                                  | Технології (попередньо)              | Пріоритет |
|--------------------------|-----------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------|---------------------------|------------------------------------------------------------------------------------------------------|--------------------------------------|-----------|
| FrontendWeb              | Веб-інтерфейс для користувача                                              | • Пошук фільмів за назвою<br>• Застосування фільтрів<br>• Перегляд карток фільмів<br>• Перехід на сторінку фільму | HTTPS + REST (JSON)      | Вхід: пошуковий запит, фільтри, клік по фільму; Вихід: відображені списки, деталі фільму, посилання | React/Vue/Angular, HTML, CSS         | MVP       |
| BackendAPI               | Серверна логіка, обробка запитів від FrontendWeb                           | • Обробка пошуку<br>• Застосування фільтрів<br>• Отримання деталей фільму<br>• Формування відповіді для FrontendWeb | HTTP REST (JSON)         | Вхід: HTTP-запити від FrontendWeb; Вихід: JSON зі списком фільмів, деталями, посиланнями           | Node.js/NestJS/Express або аналог     | MVP       |
| FilmDatabase             | Центральне сховище даних про фільми                                       | • Зберігання фільмів<br>• Зберігання метаданих (жанри, роки, теги)<br>• Зв’язки з платформами перегляду | SQL-протокол / ORM       | Вхід: запити на читання/запис від BackendAPI; Вихід: записи фільмів, пов’язані дані                 | PostgreSQL/MySQL/SQLite              | MVP       |
| ExternalPlatformsAdapter | Інтеграція з зовнішніми платформами перегляду (YouTube, Megogo, тощо)      | • Зберігання/оновлення посилань на платформи<br>• Валідація посилань<br>• Отримання списку платформ для фільму | HTTP REST (до API, якщо є) | Вхід: запити від BackendAPI з ID фільму; Вихід: список платформ перегляду з URL                     | Вбудований модуль у BackendAPI       | MVP       |
| AuthModule (опціонально) | Простий механізм авторизації для адмінського доступу/панелі керування     | • Логін адміна<br>• Мінімальний захист адмін-ендпоінтів                                          | HTTP REST + JWT/Session   | Вхід: креденшіали, токен; Вихід: статус логіну, токени, коди помилок                                | JWT/Session, bcrypt, тощо            | Future    |
| ObservabilityAndInfra    | Логування, моніторинг, деплой, CI/CD                                      | • Логування запитів/помилок<br>• Базовий моніторинг стану сервісу<br>• Автоматичний деплой         | Залежить від інструментів | Вхід: логи з BackendAPI; Вихід: дашборди/логи для розробників                                       | Docker, GitHubActions, LoggingStack  | Future    |

---

## MVP Components

MVP фокусується на:

- FrontendWeb  
- BackendAPI  
- FilmDatabase  
- ExternalPlatformsAdapter  

AuthModule та ObservabilityAndInfra можуть додаватися пізніше.
